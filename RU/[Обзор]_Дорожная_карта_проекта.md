## 1. Суть проекта

Международная финтех-компания выходит в Индию с кредитным приложением **«Helpa»**. Цель — протестировать бренд на реальных индийских заёмщиках и выдать вердикт: **запускать / доработать / отказаться**.

Методологическая база проекта описана в документе **[«Методы исследования»](Методы_исследования.md)**.

**Что тестируем:**
1. Восприятие названия — ассоциации, доверие, запоминаемость, фонетика (фреймворки: Keller CBBE, Aaker 5Bs, Brand Linguistics)
2. Прототипы приложения — удобство, понятность, безопасность
3. Страница Google Play — что привлекает, что отталкивает, что побуждает установить
4. Барьеры и страхи цифрового кредитования
5. Сравнение с конкурентами — RupeeRedee, NIRA, CASHe, mPokket, KreditBee и др.

## 3. Методология

Клиент предпочитает фокус-группы. Предлагаем **два варианта** + **параллельное анкетирование через чат-бот** для количественной базы.

> **Клиент выбирает между Вариантами B и C.** Возможен промежуточный вариант.

### Почему 70 респондентов — минимум

Бриф просит 15–24 человека, но для статистически значимых выводов нужно минимум 70. При N = 50–100 уверенно выявляются крупные и средние эффекты (непараметрические тесты). В Индии набрать такое количество реально (подтверждено Дипаншу).

### Структура: кач + кол

| Компонент | Формат | Количество | Цель |
| :--- | :--- | :--- | :--- |
| **Качественная часть** | IDI + фокус-группы | 20–32 респ. | Глубина: доверие, страхи, ассоциации, барьеры |
| **Количественная часть** | Анкетирование через чат-бот | 50–100 респ. | Статистика: масштабирование инсайтов, процентные срезы |
| **Итого** | | **~70–130 респ.** | Полноценная картина |

Анкетирование через чат-бот (Telegram / WhatsApp) запускается параллельно с интервью. Высокий отклик (знакомая среда для ЦА), визуальные стимулы встроены в поток анкеты.

### Вариант B: Микс — 1 ФГ + IDI + чат-бот (рекомендуемый)

- **1 фокус-группа** (5–6 чел., 60–90 мин.) — общие ассоциации, групповая динамика
- **15–20 IDI** (глубинные интервью 1-на-1, **40–60 мин.**) — доверие, страхи, конверсионные барьеры
- **Анкетирование через чат-бот** (50–100 чел.) — количественная база
- Итого: **~70–130 респондентов**
- Онлайн: Zoom / Google Meet (IDI и ФГ); Telegram / WhatsApp (чат-бот)
- **Почему рекомендуем:** покрываем оба формата из брифа + количественные данные сверх ожиданий; ФГ — «широкая» картина, IDI — глубина, чат-бот — статистика; воронка рекрутинга управляемая

### Вариант C: Полный цикл

- **2 фокус-группы** — Tier-1 и Tier-2 отдельно (по 5–6 участников, 60–90 мин.)
- **15–20 IDI** (глубинные интервью 1-на-1, **40–60 мин.**)
- **Расширенное анкетирование через чат-бот** (70–100 респондентов)
- Итого: **~90–130 респондентов**
- Онлайн: Zoom / Google Meet (IDI и ФГ); Telegram / WhatsApp (чат-бот)
- Дополнительно: сравнительный анализ Tier-1 vs Tier-2, стратегическая презентация клиенту

### Воронка рекрутинга по вариантам

| | Вариант B | Вариант C |
| :--- | :--- | :--- |
| Кач. часть: на сессии | 21–26 (1 ФГ + 15–20 IDI) | 25–32 (2 ФГ + 15–20 IDI) |
| Кач. часть: рекрутировать | **30–35** | **35–45** |
| Чат-бот анкетирование | 50–100 | 70–100 |
| **Общий охват** | **~70–130** | **~90–130** |

### Разбивка респондентов

- По уровню дохода (нижний / средний сегмент)
- По наличию негативного опыта с кредитными приложениями
- По tier города (Tier-1: Дели, Мумбаи, Бангалор vs Tier-2: Джайпур, Лакхнау, Пуна)

### Методологические основы

Исследование опирается на научно обоснованные фреймворки. Подробное описание — в документе **[«Методы исследования»](Методы_исследования.md)**.

| Уровень | Фреймворки | Применение в проекте |
| :--- | :--- | :--- |
| Общий подход | Mixed-methods (Creswell) | Конвергентный параллельный дизайн, триангуляция данных |
| Культура | Hofstede, GLOBE, Erin Meyer | Дизайн гайда модератора, учёт социальной желательности и стигмы долга |
| Бренд | Keller CBBE, Aaker 5Bs, Brand Linguistics | Структура тестирования названия, бенчмаркинг vs конкуренты |
| Качество данных | Lincoln & Guba, Braun & Clarke | Критерии достоверности, тематический анализ (6 фаз) |

### Верификация опыта заимствования

Используется **3-ступенчатая валидация** (подробнее — в [«Методах исследования»](Методы_исследования.md), §2.5):

1. **Распознавание приложений:** скриншоты / иконки реальных кредитных приложений (+ 2–3 несуществующих как ловушки)
2. **Описание flow:** респондент описывает процесс получения займа (последовательность шагов)
3. **Опыт погашения:** вопросы о возврате — сумма, срок, способ погашения

Респонденты из generic-панелей без верификации опыта **не допускаются**.

### Скринер (анкета отбора)

Содержание: демография (возраст, город, доход), 3-ступенчатая валидация опыта, ловушка (2–3 несуществующих приложения), исключение финтех-сотрудников, готовность к видеозвонку на 40–60 минут.

### Чат-бот анкетирование (количественная часть)

Параллельно с интервью запускается анкетирование через чат-бот (Telegram / WhatsApp):

- Структурированная анкета с визуальными стимулами (лого, скриншоты UI, макет Google Play)
