## 1. Суть проекта

Международная финтех-компания выходит в Индию с кредитным приложением **«Helpa»**. Нужно протестировать бренд на реальных индийских заёмщиках. Результат — отчёт с вердиктом: **запускать / доработать / отказаться**.

Методологическая база проекта описана в документе **[«Методы исследования»](Методы_исследования.md)** — mixed-methods дизайн, кросс-культурные фреймворки, критерии качества.

### Что именно тестируем

1. **Восприятие названия** — ассоциации, доверие, запоминаемость, фонетика (фреймворки: Keller CBBE, Aaker 5Bs, Brand Linguistics)
2. **Оценка прототипов приложения** — удобство, понятность, ощущение безопасности
3. **Оценка страницы Google Play** — что привлекает, что отталкивает, что побуждает установить
4. **Барьеры и страхи** — связанные с цифровым кредитованием
5. **Сравнение с конкурентами** — RupeeRedee, NIRA, CASHe, mPokket, KreditBee и др.

## 3. Методология

Клиент в брифе упоминает фокус-группы как предпочтительный формат. Предлагаем **два варианта** + **параллельное анкетирование через чат-бот** для количественной базы.

> **⚡ Клиент выбирает между Вариантами B и C.** Возможен промежуточный вариант.

### Почему 70 респондентов — это минимум

Бриф просит 15–24 человека, но из маленькой выборки невозможно сделать статистически значимые выводы. 70 респондентов — оптимальная минимальная планка. При N = 50–100 уверенно выявляются крупные и средние эффекты (непараметрические тесты). Учитывая плотность населения Индии и распространённость цифрового кредитования, набрать такое количество — реально (подтверждено Дипаншу).

### Структура исследования: кач + кол

| Компонент | Формат | Количество | Цель |
| :--- | :--- | :--- | :--- |
| **Качественная часть** | IDI + фокус-группы | 20–32 респ. | Глубина: доверие, страхи, ассоциации, барьеры |
| **Количественная часть** | Анкетирование через чат-бот | 50–100 респ. | Статистика: масштабирование инсайтов, процентные срезы |
| **Итого** | | **~70–130 респ.** | Полноценная картина |

Анкетирование через чат-бот (Telegram / WhatsApp) запускается параллельно с интервью. Чат-бот обеспечивает высокий отклик (знакомая среда для ЦА) и позволяет встроить визуальные стимулы прямо в поток анкеты.

### Вариант B: Микс — 1 фокус-группа + IDI + чат-бот анкетирование (рекомендуемый)

- **1 фокус-группа** (5–6 чел., 60–90 мин.) — общие ассоциации, групповая динамика
- **15–20 IDI** (глубинные интервью 1-на-1, **40–60 мин.**) — доверие, страхи, негативный опыт, конверсионные барьеры
- **Анкетирование через чат-бот** (50–100 чел.) — количественная база
- Итого: **~70–130 респондентов**
- Онлайн, через Zoom / Google Meet (IDI и ФГ); Telegram / WhatsApp (чат-бот)
- Анастасия справляется с полным объёмом (тянет 30–40 респондентов)

**Почему рекомендуем Вариант B:**
- Покрываем оба формата из брифа + даём клиенту количественные данные (бонус сверх ожиданий)
- Фокус-группа — «широкая» картина, IDI — глубина, чат-бот — статистика
- Воронка рекрутинга более управляемая
- Клиент сам указал, что IDI особенно полезны для тем доверия и страхов

### Вариант C: Полный цикл

- **2 фокус-группы** — отдельно для Tier-1 и Tier-2 (по 5–6 участников каждая, 60–90 мин.)
- **15–20 IDI** (глубинные интервью 1-на-1, **40–60 мин.**)
- **Расширенное анкетирование через чат-бот** (70–100 респондентов)
- Итого: **~90–130 респондентов**
- Онлайн, через Zoom / Google Meet (IDI и ФГ); Telegram / WhatsApp (чат-бот)

**Что входит дополнительно (сверх Варианта B):**
- Сравнительный анализ восприятия бренда по сегментам (Tier-1 vs Tier-2)
- Стратегическая презентация результатов с командой клиента

### Воронка рекрутинга по вариантам

| | Вариант B | Вариант C |
| :--- | :--- | :--- |
| Кач. часть: на сессии | 21–26 (1 ФГ + 15–20 IDI) | 25–32 (2 ФГ + 15–20 IDI) |
| Кач. часть: рекрутировать | **30–35** | **35–45** |
| Чат-бот анкетирование | 50–100 | 70–100 |
| **Общий охват** | **~70–130** | **~90–130** |

Набрать заёмщиков в Индии — не проблема. Дипаншу (Deepanshu) подтвердил: сам брал кредиты, и в его окружении это массовое явление. Цифровое кредитование в Индии — часть повседневной жизни для нашей ЦА.

### Разбивка респондентов

- По уровню дохода (нижний / средний сегмент — уточнить по ТА из брифа)
- По наличию негативного опыта с кредитными приложениями
- По tier города (Tier-1: Дели, Мумбаи, Бангалор vs Tier-2: Джайпур, Лакхнау, Пуна)

### Методологические основы

Исследование опирается на научно обоснованные фреймворки. Подробное описание — в документе **[«Методы исследования»](Методы_исследования.md)**.

| Уровень | Фреймворки | Применение в проекте |
| :--- | :--- | :--- |
| Общий подход | Mixed-methods (Creswell) | Конвергентный параллельный дизайн, триангуляция данных |
| Культура | Hofstede, GLOBE, Erin Meyer | Дизайн гайда модератора, учёт социальной желательности и стигмы долга |
| Бренд | Keller CBBE, Aaker 5Bs, Brand Linguistics | Структура тестирования названия, бенчмаркинг vs конкуренты |
| Качество данных | Lincoln & Guba, Braun & Clarke | Критерии достоверности, тематический анализ (6 фаз) |

### Верификация опыта заимствования

Бриф жёстко требует подтверждённый опыт. Просто «галочку в анкете» — недостаточно. Используется **3-ступенчатая валидация** (подробнее — в [«Методах исследования»](Методы_исследования.md), §2.5):

1. **Распознавание приложений:** показываем скриншоты / иконки реальных кредитных приложений (+ 2–3 несуществующих как ловушки)
2. **Описание flow:** респондент описывает процесс получения займа (последовательность шагов)
3. **Опыт погашения:** вопросы об опыте возврата — сумма, срок, способ погашения

Респонденты из generic-панелей без верификации опыта **не допускаются**.

### Скринер (анкета отбора)

Что должно быть:

1. Демография: возраст, город, доход
2. Подтверждение опыта: 3-ступенчатая валидация (распознавание приложений, описание flow, опыт погашения)
3. Ловушка: включить 2–3 несуществующих названия приложений в список
4. Исключение: «Работаете ли вы в финансовом / технологическом секторе?»
5. Готовность к видеозвонку на 40–60 минут

### Чат-бот анкетирование (количественная часть)

Параллельно с глубинными интервью запускается анкетирование через чат-бот (Telegram / WhatsApp):

- Структурированная анкета с визуальными стимулами (лого, скриншоты UI, макет Google Play), встроенными прямо в поток
- Пошаговый проход по блокам: бренд → UI → Google Play → итог
- Логические ветвления и скип-логика
- Шкалы семантического дифференциала (доверие, дружелюбие, легитимность), шкалы Лайкерта, ранжирование ассоциаций
- Перед допуском к анкете чат-бот проводит скрининг (демография + косвенная валидация опыта)
- Цель: набрать 50–100 ответов для количественной базы

## 7. Deliverables — что клиент получит

1. **Аналитический отчёт** — выводы и рекомендации: восприятие бренда, барьеры, точки роста, бенчмаркинг vs конкуренты (по модели Aaker 5Bs)
2. **Чёткая рекомендация** — формализованная: использовать как есть / адаптировать / отклонить — с обоснованием и конкретными шагами по каждому сценарию
3. **Verbatim-цитаты** — ключевые цитаты респондентов с переводом на английский и контекстом (город, доход, пол, метод)
4. **Количественные срезы** — по результатам чат-бот анкетирования (семантический дифференциал, шкалы Лайкерта, непараметрические тесты)
5. **Видеозаписи** — всех интервью и фокус-групп
6. **Карта восприятия бренда** *(опционально)* — визуализация: доверие × дружелюбие × легитимность
7. **Варианты фонетических адаптаций названия** *(опционально)* — только если спонтанно предложены респондентами
