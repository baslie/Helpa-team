## 1. Суть проекта

Финтех-компания **Cyberbird** выходит в Индию с кредитным приложением **«Helpa»**. Нужно протестировать бренд на реальных индийских заёмщиках. Результат — отчёт с вердиктом: **запускать / доработать / отказаться**.

### Что именно тестируем

1. **Восприятие названия** — ассоциации, доверие, запоминаемость
2. **Оценка прототипов приложения** — удобство, понятность, ощущение безопасности
3. **Оценка страницы Google Play** — что привлекает, что отталкивает, что побуждает установить
4. **Барьеры и страхи** — связанные с цифровым кредитованием
5. **Сравнение с конкурентами** — KreditBee, MoneyTap, CASHe и др.

## 3. Методология

Клиент в брифе упоминает фокус-группы как предпочтительный формат. Предлагаем **три варианта** + **параллельное онлайн-анкетирование** для количественной базы.

> **⚡ Клиент выбирает между Вариантами B и C.** Возможен промежуточный вариант.

### Почему 50 респондентов — это минимум

Бриф просит 15–24 человека, но из маленькой выборки невозможно сделать статистически значимые выводы. 50 респондентов — оптимальная минимальная планка. Учитывая плотность населения Индии и распространённость цифрового кредитования, набрать такое количество — реально (подтверждено Дипаншу).

### Структура исследования: кач + кол

| Компонент | Формат | Количество | Цель |
| :--- | :--- | :--- | :--- |
| **Качественная часть** | IDI + фокус-группы | 18–24 респ. | Глубина: доверие, страхи, ассоциации, барьеры |
| **Количественная часть** | Онлайн-анкетирование | 30–50 респ. | Статистика: масштабирование инсайтов, процентные срезы |
| **Итого** | | **~50+ респ.** | Полноценная картина |

Анкетирование запускается параллельно с интервью. Конверсия будет не очень высокая, но ответы приходят — подтверждено опытом мониторингов удовлетворённости.

### Вариант A: Фокус-группы + анкетирование

- 3–4 онлайн фокус-группы по 5–6 участников (18–24 чел.) + онлайн-анкета (30–50 чел.)
- Длительность ФГ: 90–120 минут каждая
- Через Zoom / Google Meet

**Плюсы:** Соответствует предпочтению клиента. Групповая динамика — респонденты подхватывают идеи друг друга.

**Минусы и риски:**
- Воронка шире в 2–2,5 раза. Для 24 участников ФГ нужно рекрутировать 40–50 (неявка ~20–30%).
- Сложнее организовать: собрать 5–6 человек в одно время + часовые пояса.
- Чувствительные темы раскрываются хуже в группе.

### Вариант B: Микс — 1 фокус-группа + IDI + анкетирование (рекомендуемый)

- **1 фокус-группа** (5–6 чел.) — общие ассоциации, групповая динамика
- **15–20 IDI** (глубинные интервью 1-на-1) — доверие, страхи, негативный опыт, конверсионные барьеры
- **Онлайн-анкета** (30–50 чел.) — количественная база
- Длительность IDI: **30–45 минут**
- Онлайн, через Zoom / Google Meet
- Анастасия справляется с полным объёмом (тянет 30–40 респондентов)

**Почему рекомендуем Вариант B:**
- Покрываем оба формата из брифа + даём клиенту количественные данные (бонус сверх ожиданий)
- Фокус-группа — «широкая» картина, IDI — глубина, анкета — статистика
- Воронка рекрутинга более управляемая
- Клиент сам указал, что IDI особенно полезны для тем доверия и страхов

### Вариант C: Полный цикл

- **2 фокус-группы** — отдельно для Tier-1 и Tier-2 (по 5–6 участников каждая)
- **15–20 IDI** (глубинные интервью 1-на-1)
- **Расширенное онлайн-анкетирование** (70–100 респондентов)
- Итого: **~90–130 респондентов**
- Длительность IDI: **30–45 минут**, ФГ: 90–120 минут
- Онлайн, через Zoom / Google Meet

**Что входит дополнительно (сверх Варианта B):**
- Сравнительный анализ восприятия бренда по сегментам (Tier-1 vs Tier-2)
- Стратегическая презентация результатов с командой клиента

### Воронка рекрутинга по вариантам

| | Вариант A | Вариант B | Вариант C |
| :--- | :--- | :--- | :--- |
| Кач. часть: на сессии | 18–24 | 21–26 (1 ФГ + 15–20 IDI) | 25–32 (2 ФГ + 15–20 IDI) |
| Кач. часть: рекрутировать | **40–50** | **30–35** | **35–45** |
| Анкетирование | 30–50 | 30–50 | 70–100 |
| **Общий охват** | **~50–70** | **~50–75** | **~90–130** |

Набрать заёмщиков в Индии — не проблема. Дипаншу (Deepanshu) подтвердил: сам брал кредиты, и в его окружении это массовое явление. Цифровое кредитование в Индии — часть повседневной жизни для нашей ЦА.

### Разбивка респондентов

- По уровню дохода (нижний / средний сегмент — уточнить по ТА из брифа)
- По наличию негативного опыта с кредитными приложениями
- По tier города (Tier-1: Дели, Мумбаи, Бангалор vs Tier-2: Джайпур, Лакхнау, Пуна)

| Инструмент | Для чего |
| :--- | :--- |
| **Google Meet** (встроенная запись) | Основной вариант — запись видео + аудио. Файлы в Google Drive. Нет лимита по времени |
| **OBS Studio** | Бэкап-запись экрана на случай сбоя Meet |
| **Zoom** (бесплатный план) | Альтернатива; для IDI 30–45 мин. бесплатного плана может хватить |

**Рекомендация:** Google Meet как основной + OBS как подстраховка.

### Транскрибация и анализ

Респонденты могут быть говорливыми, акцент может затруднять понимание — нужен инструмент с хорошим AI-движком.

| Инструмент | Плюсы | Минусы |
| :--- | :--- | :--- |
| **Otter.ai** (Pro) | AI-саммаризация, выделение ключевых моментов, поиск по тексту, интеграция с Zoom/Meet | Может плохо работать с индийским английским |
| **Fireflies.ai** (Pro) | AI-саммаризация, выявление тем и action items, хороший для акцентов | — |
| **Trint** | Отличная точность, поддержка акцентов, мощный редактор | — |
| **Tactiq** | Живая транскрибация в Google Meet, AI-саммаризация | Базовый план ограничен |

**Рекомендация:** Otter.ai Pro — оптимальное соотношение возможностей. Если нужна максимальная точность с акцентами — Trint.

### Скринер (лид-форма)

| Инструмент | Комментарий |
| :--- | :--- |
| **Google Forms** | Простой, достаточный для скрининга |
| **Typeform** | Красивее, лучше UX, skip-логика |
| **Лид-формы Facebook/Instagram** | Респондент заполняет прямо в соцсети — минимальное трение. Настраивает Артём |

**Рекомендация:** лид-формы в Facebook/Instagram (первичный фильтр, настраивает Артём) → Google Forms для расширенного скринера (вторичная верификация).

### Организация и хранение

- **Google Drive** — все записи, транскрипты, документы

- Живёт в Индии (город Tier-1: Дели, Мумбаи, Бангалор — или Tier-2: Джайпур, Лакхнау, Пуна и др.)
- Реально пользовался кредитными приложениями (подтверждено, не на словах)

- Не работает в финтехе
- Рекрутировать с запасом +25–30% (неявка у индийских респондентов ~20–30%)

| :--- | :--- |

**Способ оплаты: UPI (обязательно!)** — не подарочные карты. UPI — основная платёжная инфраструктура этой аудитории. Для оплаты через UPI — помощь Дипаншу (есть индийский банковский счёт).

### Верификация опыта заимствования

Бриф жёстко требует подтверждённый опыт. Просто «галочку в анкете» — недостаточно.

1. **Скринер с ловушками:** включить 2–3 несуществующих названия кредитных приложений
2. **Конкретные вопросы:** «Какую сумму брали? Какой был срок? Какой процент?»
3. **Скриншоты:** попросить показать кредитное приложение на скрининг-звонке
4. **Double screening:** короткий телефонный/видео-звонок перед основным интервью

### Скринер (анкета отбора) — нужен ASAP

Что должно быть:

1. Демография: возраст, город, доход
2. Подтверждение опыта: «Какими кредитными приложениями пользовались? Назовите. Какую сумму брали? Когда последний раз?»
3. Ловушка: включить 2–3 несуществующих названия приложений в список
4. Исключение: «Работаете ли вы в финансовом / технологическом секторе?»
5. Готовность к видеозвонку на 30–45 минут

### Онлайн-анкета (количественная часть) — отдельный документ

Параллельно с глубинными интервью запускается онлайн-анкетирование:

- Развёрнутый опросник с вопросами из раздела 5 брифа (бренд, доверие, ассоциации)
- Респонденты заполняют самостоятельно (ссылка через WhatsApp / соцсети)
- Конверсия будет невысокая, но ответы приходят (подтверждено опытом Маргариты)
- Цель: набрать 30–50 ответов для количественной базы
- Инструмент: Google Forms / Typeform
